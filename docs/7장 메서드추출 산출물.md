# 7장 - 메서드 추출 산출물 (Long Method + 깊은 중첩 제거)

`processApproval`의 긴 메서드(약 86줄)·깊은 중첩(4~5단 if)을 **가드 절(Guard Clause) + 메서드 추출(Extract Method)**로 분해.
**외부 시그니처·동작·DB 저장값 100% 보존** (순수 리팩토링).

---

## 수정 (1개)

| 파일 | 변경 내용 |
|---|---|
| `approval/ApprovalService.java` | `processApproval` 분해 - 가드 절 + action별 private 메서드 위임 |

신규 파일 없음 (메서드 추출은 기존 클래스 내부 구조만 변경).

---

## Before → After

**Before**: `processApproval` 한 메서드에 상신/승인/반려/취소가 중첩 if로 뭉쳐 있음 (86줄, 4~5단 중첩)

**After**: 진입점은 얇게, 분기는 private 메서드로 위임
```java
public void processApproval(Long id, Long userId, int action, String reason)
  // 조회 + 가드 절(null 등 조기 반환)
  // action별 private 위임:
  private void submit(Approval approval, ...)   // action 1 상신
  private void approve(Approval approval, ...)  // action 2 승인
  private void reject(Approval approval, ...)   // action 3 반려
  private void cancel(Approval approval, ...)   // action 9 취소
```

식별자도 약어 없이 의미 있는 이름(`approval`, `user`)으로 유지.

---

## 보존한 것 (불변 규칙)

- `processApproval(id, userId, action, reason)` **외부 시그니처 그대로**
- 상태 전이 결과·조용한 무시 등 **동작 100% 보존**
- **특성화 테스트 6개: 한 글자도 변경 없이 green** (해시 동일) = 동작 안 바뀜 증거
